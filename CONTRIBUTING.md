## 🚀 2. How to Interface with the Core

### 🐛 A. Bug Anomalies (Issue Reporting)
If the neural architecture fails, we need precise telemetry to patch it.
- **Use the `[ANOMALY]` tag** in your issue title.
- **Provide the Stack Trace:** We need terminal logs, browser console errors, and exact steps to reproduce.
- **Visuals:** Screenshots or GIF recordings are highly encouraged.

### 💡 B. Neural Enhancements (Feature Requests)
Have an idea for a new AI Agent? A better Pomodoro algorithm?
- **Use the `[ENHANCEMENT]` tag** in your issue title.
- **Justify the Logic:** Explain exactly *why* this feature reduces cognitive friction for the user.
- **Provide a Mockup:** If it's a UI change, provide a rough visual structure.

## ⚙️ 3. The Pull Request (PR) Pipeline

To integrate your code into the `main` architecture, you must survive the deployment pipeline:

| Phase | Action Required | Standard to Meet |
| :--- | :--- | :--- |
| **01. Fork & Branch** | Branch from `main`. | Format: `feature/your-upgrade` or `patch/issue-id`. |

## 🎨 4. Architectural Styleguide

> [!IMPORTANT]
> This project adheres to strict design and logic constraints.

- **Frontend (Vite + React 19):** Use Tailwind CSS v4. Avoid arbitrary utility classes if a theme variable exists. Use `Framer Motion` for all transitions—static UI is not permitted.
- **Backend (Node + Prisma):** Keep controllers purely logical. Do not mix database logic inside routing files.
- **AI Telemetry:** Any modifications to the `BlazeFace` TensorFlow.js model MUST remain 100% client-side. **Uploading video data to the cloud is strictly forbidden.**
